## Schemas

### Turn

```ts
export interface NoteTurn {
  turnId: string;
  artifactId: string;
  phase: "Observe" | "Propose" | "Counter" | "Synthesize" | "Handoff";
  role:
    | "scribe"
    | "critic"
    | "synthesizer"
    | "human-reviser"
    | "router"
    | "implementation-engine";
  agentId: string;
  confidence: number; // 0.0 - 1.0
  assumptions: string[];
  openQuestions: string[];
  decision: string;
  evidence: string[];
  requestedAction: "commit" | "escalate" | "ask-human" | "merge" | "defer";
  replyTo?: string;
  expiresAt?: string;
  createdAt: string;
  updatedAt: string;
}


```

## Validation rules

For every request, validate:

1. `confidence` is between `0` and `1`.
1. `assumptions.length >= 1`.
1. `openQuestions.length >= 1`.
1. `phase` transitions follow `Observe -> Propose -> Counter -> Synthesize -> Handoff`.
1. `requestedAction` is compatible with `phase`:
   - `Observe`: `ask-human` or `escalate`
   - `Propose`/`Counter`/`Synthesize`: `defer` or `handoff`
   - `Handoff`: `commit` or `escalate`

## Suggested data stores

- Primary: append-only relational table (immutable turns with `replyTo` references).
- Indexes:
  - `(artifactId, phase)`
  - `(artifactId, replyTo)`
  - `(requestedAction, createdAt)`
- Secondary store: vector index on turn text for semantic recall during next-turn drafting.

## Error behavior

Use strict failures for protocol violations and soft retries for transient
transient infra failures.

If a turn request fails schema validation, return:

```json text


{
  "ok": false,
  "error": {
    "code": "TURN_VALIDATION_ERROR",
    "message": "Assumptions and open questions are required before handoff."
  }
}


``` text



For unsafe confidence mismatches, return:



```json text


{
  "ok": false,
  "error": {
    "code": "INVALID_CONFIDENCE",
    "message": "Confidence must be within [0.0, 1.0]."
  }
}


```

## Success criteria for API parity

By the end of MVP, the API should:

- persist every accepted turn
- support replay of full thread history
- expose unresolved question counts per artifact
- fail openQuestion rules deterministically
- produce audit exports in JSONL
