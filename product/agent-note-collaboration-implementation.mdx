This page shows a concrete implementation path for the note-based collaboration
engine in `src/lib/agent-note-collab`.

## Current implementation

- `src/lib/agent-note-collab/turn-log.ts`: shared types and validation rules
- `src/lib/agent-note-collab/route.ts`: routing logic for accept/retry/escalate
- `src/lib/agent-note-collab/store.ts`: in-memory ledger and submission flow
- `src/lib/agent-note-collab/index.ts`: consolidated exports

## Example: creating and submitting a turn

```ts
import {
  InMemoryTurnLedger,
  createTurnRecord,
  formatTurnError,
} from "@lib/agent-note-collab";

const ledger = new InMemoryTurnLedger();

try {
  const result = ledger.submitTurn({
    turnId: "",
    artifactId: "docs/agent-note-collaboration.md",
    phase: "Observe",
    role: "scribe",
    agentId: "agent-alpha",
    confidence: 0.82,
    assumptions: ["Context parser output is fresh."],
    openQuestions: ["What is the target confidence threshold?"],
    decision: "Start with artifact-level handoff for draft proposal.",
    evidence: ["agent-note-collaboration.md#section-observation"],
    requestedAction: "escalate",
  });

  if (!result.ok) {
    console.error("Turn rejected:", result.errors);
  } else {
    console.log("Stored:", result.turn.turnId, "nextPhase:", result.nextPhase);
  }
} catch (error) {
  const failure = formatTurnError(error);
  console.error("Validation failed:", failure.errors);
}


```

## Suggested next engineering step

- replace in-memory store with a durable persistence adapter (PostgreSQL or
  document DB),
- add a typed API layer that calls `submitTurn`,
- add authorization so only approved roles can write to sensitive artifacts.
