## What You'll Need

Before you start, make sure you have:

- **Node.js** 18+ (we recommend 20 LTS)
- **pnpm** — our package manager for JavaScript/TypeScript
  (*never* use npm or yarn)
- **uv** — our package manager for Python (*never* use pip
  or conda)
- **Docker** — for running the full service stack
- **Git** — obviously

<Warning>
  **Package manager rules are non-negotiable.** Always use `pnpm` for Node.js
  and `uv` for Python. Never use `npm`, `yarn`, `pip`, or `conda`.
</Warning>

## Quick Setup

<Steps>
  <Step title="Clone the Repository">
  ```bash
  git clone https://github.com/pixelated-empathy/pixelated.git
  cd pixelated
  ```
  </Step>

  <Step title="Install Dependencies">
  ```bash
  # Node dependencies
  pnpm install

# Python dependencies

uv install

````
</Step>

<Step title="Start the Dev Server">
```bash
pnpm dev
````

That gets you the frontend running at
`http://localhost:4321`.

  </Step>
</Steps>

## Running Everything

The full Pixelated Empathy stack includes multiple services.
To start them all:

```bash
pnpm dev:all-services
```

This spins up:

- **Frontend** — Astro dev server (port 4321)
- **AI Service** — FastAPI emotional intelligence engine
- **Worker** — Background processing
- **WebSocket Server** — Real-time conversation updates

Alternatively, use Docker Compose for a production-like
environment:

```bash
pnpm docker:up
```

## Environment Variables

Copy the example env file and fill in your values:

```bash
cp .env.example .env.local
```

Key variables you'll need:

```bash
NODE_ENV=development
PORT=4321
MEM0_API_KEY=your-mem0-api-key
```

<Warning>**Never commit `.env.local`.** It's already in `.gitignore`.</Warning>

## Path Aliases

We use path aliases to keep imports clean. Instead of:

```typescript
import { EmotionAnalyzer } from '../../../lib/ai/emotion-analyzer'
```

You write:

```typescript
import { EmotionAnalyzer } from '@/lib/ai/emotion-analyzer'
```

Available aliases:

| Alias           | Maps To              |
| :-------------- | :------------------- |
| `@/*`           | `./src/*`            |
| `@lib/*`        | `./src/lib/*`        |
| `@components/*` | `./src/components/*` |
| `@utils/*`      | `./src/utils/*`      |
| `@types/*`      | `./src/types/*`      |
